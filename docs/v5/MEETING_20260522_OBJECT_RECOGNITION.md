## 핵심 지적 사항

### 1. 96.6% 수치는 아직 "basket을 본다"의 증거가 아님

> "객체 인식 여부가 확인되지 않은 상태에서 액션 헤드 정확도만으로 결론 내리면 안 된다."

- 96.6%가 val set 기준 → val set을 최종 성능 평가에 쓰는 것 자체가 부적절
- 방향이 맞아도 basket을 보고 맞은 건지, 복도 패턴 암기인지 여전히 불분명
- Track 2 (zero-shot probe)도 val set 기준이라 동일한 한계

### 2. LoRA 적용 전후 비교가 없다

> "Frozen CLIP만으로도 96.6%가 나왔다면 LoRA가 뭘 했는지 모르는 것 아닌가?"

현황:
| 모델 | 정확도 | 비고 |
|------|--------|------|
| Frozen CLIP + logistic regression | 96.6% | **비교 베이스라인** |
| Stage 1 v2 (CLIP LoRA 학습 후) | 98.1% | **+1.5%p — 차이 미미** |

→ LoRA의 실질 기여도 불분명. 이 비교가 실험 설계에 빠져 있었음.

### 3. 올바른 테스트 케이스가 없다

교수님이 제시한 검증 방법:
- **다른 객체 입력**: basket 대신 전혀 다른 물체를 줬을 때 이상한 행동을 하는가?
- **마스킹**: basket 영역을 가렸을 때 결과가 달라지는가? ← Track 3로 부분 검증됨
- **방향 정보 없이 목표만**: "tracking basket" 지시만 줬을 때 올바르게 움직이는가?

### 4. 사전학습 객체 목록 미파악

> "RoboVLM 사전학습 때 어떤 객체로 학습됐는지 알아야 한다."

- 하이레벨 LoRA(18~23 레이어) 학습 후 기존 객체 인식 성능이 **저하되어서는 안 된다**
- 새 객체(basket) 인식 + 기존 객체 인식 유지 = 올바른 학습
- 현재 기존 객체 성능 저하 여부 미확인
