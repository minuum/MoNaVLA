## 핵심 실패 패턴

### 1. PM 자체가 무너짐

Exp18의 PM은 `27.62% (50/181)`로 매우 낮았습니다.  
validation loss는 `1.325`까지 내려갔지만, 실제 action matching은 따라오지 못했습니다.

### 2. FORWARD가 전부 FWD+R로 붕괴

PM confusion 기준 핵심 오류:

- `FORWARD`: `95 / 95`가 전부 `FWD+R`로 오분류
- `RIGHT`: `10 / 10` 모두 실패
- `FWD+L`: `4 / 4` 모두 실패

즉 Exp18은 text embedding fusion을 넣었음에도, 실제 정책은 **우측 대각선 행동으로 쏠리는 collapse**를 보였습니다.

### 3. Closed-loop에서는 9개 path 중 1개만 성공

Exp18 closed-loop path 결과:

- 성공: `center_right`만 성공
- 실패: `center_straight`, `center_left`, `left_straight`, `left_left`, `left_right`, `right_straight`, `right_left`, `right_right`

이는 PM failure가 rollout에서 그대로 누적된 전형적인 사례입니다.

---

## 중간 결론

Exp18은 다음 두 가지를 동시에 보여줍니다.

1. **validation loss 개선은 deployment gain을 보장하지 않는다**
2. **text embedding fusion도 현재 end-to-end instability를 해결하지 못했다**

따라서 현재 결론은 더 강해졌습니다.

> End-to-end VLM branch는 closed-loop gate를 통과하기 전까지 mainline이 될 수 없습니다.  
> 현재 strongest practical baseline은 여전히 **Exp14 Step2 (66.7%)** 입니다.
