§2

PaliGemma의 핵심 메커니즘 — 왜 효과적인가

SigLIP + Gemma + loc 토큰 + LoRA 구조가 grounding에 왜 강력한가

🏗️ 아키텍처 흐름

입력: RGB 이미지 (224×224) + 텍스트 쿼리 "detect gray basket"

[1] SigLIP-So400m (Vision Encoder)
   → 이미지를 256개 패치 임베딩으로 변환
   → Sigmoid-based contrastive learning으로 사전학습
     ※ CLIP과 달리 softmax가 아닌 sigmoid loss → 배치 크기에 무관, 더 안정적

[2] Linear Projection (Bridge)
   → 이미지 패치 임베딩을 Gemma 토큰 차원(2048)으로 변환
   → "소프트 토큰"으로 텍스트 시퀀스와 concat

[3] Gemma-2B (Language Decoder) + Full Block Attention
   → [이미지 토큰 256개] + [텍스트 토큰 N개] 전체에 attention
   → "detect gray basket"이 256개 이미지 패치 전체에 cross-attend
   → 다음 토큰을 autoregressive하게 생성

[4] loc 토큰 출력
   → vocab에 추가된 1024개 특수 토큰: <loc0000> ~ <loc1023>
   → [y1, x1, y2, x2] 순서로 4개 출력 → bbox 완성
   → 없으면 <eos>만 출력 (탐지 거부)

출력: <loc0462><loc0354><loc0862><loc0597> gray basket<eos>
   → cx = (354+597)/2/1023 = 0.464 → Action MLP 입력

💡 왜 Grounding에 효과적인가 — 3가지 메커니즘

① Text-Conditional Attention
텍스트 "gray basket"이 256개 이미지 패치에 full block attention으로 attend

→ 의미를 먼저 이해하고 (recognize) 위치를 결정 (locate)

증거: Exp57 — 동일 이미지에서 "red ball"로 바꾸면 <eos> 출력
② loc 토큰 설계
bbox를 언어처럼 생성.
이미지 좌표계를 0~1023 정수로 균일 분할

→ 연속 좌표 회귀(Faster R-CNN 방식)보다 언어모델에 자연스러움

출처: PaliGemma §3.2 "Vocabulary extension for location tokens"
③ SigLIP 사전학습
CLIP보다 효율적인 sigmoid loss로 image-text 정렬 학습

→ 소규모 LoRA (1.9M params / 2.9B 전체) 만으로도 빠른 도메인 적응

Exp57 결과: zero-shot 65% → LoRA 20 epoch → 100%

PM이 학습 · 추론 · 데이터 수집에서 하는 역할

각 단계에서 PaliGemma LoRA의 구체적 기능

📦 데이터 수집 단계

  • Pseudo-label 생성기: Exp58에서 V4 brown pot 데이터 524 에피소드를 HSV 없이 자동 주석
  • "detect brown pot" 쿼리로 각 프레임 bbox 자동 생성
  • 수동 주석 대비 시간 ~90% 절감
  • Hard Negative 데이터셋 구축의 핵심 도구
# Exp58: 524 에피소드 자동 주석
✅ brown pot pseudo-label
✅ red ball pseudo-label
✅ person pseudo-label

🎓 학습 단계

  • backbone 2.9B 파라미터 동결(frozen)
  • LoRA adapter 1.9M params만 업데이트
  • 입력: (image, "detect {obj}") → 출력: <locNNNN>×4
  • Hard Negative 포함 학습 → FP 0% 달성
  • 20 epoch, A5000 24GB, ~8시간
# Exp59 학습 결과
hit=95% FP=0.0%
trainable: 1,916,928
total: 2,925,383,408

⚡ 추론 단계 (실시간)

  • 로봇 카메라 프레임 수신 → "detect gray basket"
  • PaliGemma가 loc 토큰으로 bbox 반환
  • cx = (x1+x2)/2/W → [-1,1] 정규화
  • cx → Action MLP → [linear_x, linear_y]
  • bbox 없으면 (<eos>) → STOP 신호
# 실시간 파이프라인
image → PM → cx=0.46
→ MLP → forward action
핵심 역할 요약: PM(PaliGemma LoRA)은 단순한 물체 탐지기가 아니라, 텍스트 목표를 공간 좌표로 변환하는 Goal-Conditioned Grounding 모듈이다. 텍스트 쿼리가 바뀌면 자동으로 다른 물체를 탐지하고, 다른 cx를 반환하여 다른 action을 유도한다. 이것이 바로 "Mobile VLA"의 핵심 — f(image, text_goal) → bbox → action

🔄 Open-Loop vs Closed-Loop — 근본적 차이

Open-Loop (기존 PM 지표)
GT 프레임에 모델 출력 비교
→ PM (Perfect Match Rate)

장점 빠름, 재현 가능, 실제 로봇 불필요
단점 에러 누적 효과 미측정

프레임 1의 실수가 프레임 2에
영향 안 줌 → 낙관적 평가

우리: EXP-17 PM=94.72%
Closed-Loop (CL — Gold Standard)
실제 에피소드를 처음부터 끝까지 실행
→ Success Rate

장점 에러 누적, 시각 피드백, 실제 성능 측정
단점 시간/비용, 재현성 어려움

이전 action의 결과가 다음 observation에
반영 → 현실적 평가

우리: Exp59 CL=4.5% (분석 중)
중요한 발견 (OpenVLA §5.3 참조): "Offline imitation accuracy와 CL success rate 간 상관관계가 낮다." 이것이 우리가 PM 94.72%에도 불구하고 CL 평가를 독립적으로 수행하는 이유다. Kim et al., OpenVLA 2024
§5

CL이 학습 · 추론 · 데이터 수집에서 하는 역할

각 단계에서 Closed-Loop 평가가 어떤 기능을 하는가

📦 데이터 수집

  • CL 실패 케이스 분석 → 어떤 시나리오가 부족한지 파악
  • 예: CL에서 left_straight FPE=0.654m → 출발 지점 다양성 부족 발견
  • 실패 에피소드 패턴을 분석해 추가 수집 우선순위 결정
  • Hard Negative 필요성을 CL 오탐 분석으로 발견 (→ Exp59)

🎓 학습 최적화

  • CL gap을 역산해 분포 불일치 원인 파악
  • PM=94.72% vs CL=4.5% → MLP가 VLM bbox 분포에 OOD
  • → Stage2 MLP 재학습: VLM bbox 노이즈 Data Augmentation
  • EMA smoothing (α=0.5) 실험 중

⚡ 추론 검증

  • 실제 로봇/시뮬레이션에서 전체 에피소드 실행
  • 22개 validation 에피소드 × Exp59 모델 연결
  • FPE (Final Position Error) ≤ 0.5m = 성공 기준
  • CL success rate = 논문의 최종 성능 지표
현재 상태 (Exp59 CL, 2026-05-29):
• Grounding 단계 성공률 = 98% (basket을 거의 놓치지 않음)
• CL 최종 성공률 = 4.5% — Stage2 MLP가 VLM bbox의 미세 노이즈에 민감하게 반응하는 OOD 문제
• left_straight FPE=0.654m, right_right FPE=0.575m — 성공 기준(0.5m) 근접
• 현재 EMA smoothing (α=0.5) + Joint Training 해결책 실험 진행 중

§7

MoNaVLA에서의 PM + CL 전체 흐름

두 개념이 시스템 안에서 어떻게 연결되는가

🔗 PM → Grounding → Action → CL 파이프라인

────────── 수집 단계 ──────────
ROS 로봇 → HDF5 에피소드 (이미지 + GT action)
PaliGemma pseudo-label → Hard Negative 데이터 (Exp58)

────────── 학습 단계 ──────────
Stage 1: PaliGemma LoRA 학습 (Exp57/Exp59)
  → PM(hit rate) 100% 달성
  → FP 0% (Hard Negative 학습)

Stage 2: Action MLP 학습 (Exp54)
  → cx/cy → [linear_x, linear_y]
  → 문제: HSV bbox 분포로만 학습 → VLM bbox에 OOD

────────── Open-Loop 평가 (PM 지표) ──────────
GT 프레임에 모델 출력 비교 → PM=94.72% (EXP-17)
                                 Grounding hit=100% (Exp57)

────────── Closed-Loop 평가 (CL 지표) ──────────
실제 에피소드 처음부터 실행 (22 val episodes)
  → Grounding 98% 성공 (basket 탐지 자체는 OK)
  → MLP OOD → CL 4.5% ← 현재 병목

────────── 해결 방향 ──────────
① Joint Training: VLM bbox 노이즈 분포를 MLP 학습에 포함
② EMA(α=0.5): cx/cy 시계열 스무딩으로 drift 감소
③ End-to-End: PaliGemma LoRA + MLP 동시 fine-tuning (미래)
교수님 질문 연결:
"다른 물체 넣으면 다른 행동해야" → PM(grounding hit rate)이 텍스트 조건부 탐지를 증명하고, CL(closed-loop success rate)이 그것이 실제 주행으로 이어지는지 검증한다.

PM=100% 는 "모델이 올바른 것을 보고 있다"는 증거.
CL=4.5% 는 "보는 것을 행동으로 변환하는 인터페이스에 문제"가 있다는 진단 → 해결 방향이 명확함.
