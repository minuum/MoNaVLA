## 🎯 정리 우선순위 (삭제하지 않고 리스트업만 함)

### [우선순위 1] 구버전 실험 결과 (가장 안전함)
- **대상:** `third_party/RoboVLMs/runs/v3_classification` (**63G**)
  - V3-Exp05, Exp06, Exp07 등 이미 종료된 실험의 중간 체크포인트들입니다.
  - 각 체크포인트가 **8.2GB**씩 차지하고 있습니다.
  - 최신/최상위 결과(best)만 남기고 나머지는 정리 가능합니다.
- **대상:** `third_party/RoboVLMs/runs/exp_v2_series` (**15G**)
  - 매우 오래된 V2 시리즈 실험 데이터입니다.

### [우선순위 2] 현재 학습 중인 V4 중간 결과
- **대상:** `runs/v4_nav/kosmos/mobile_vla_v4_exp01/.../` (**33G**)
  - 현재 Epoch 02, 03, 04의 체크포인트가 각각 **8.2GB**씩 생성되어 있습니다.
  - 학습이 안정권에 접어들면 `last.ckpt`와 가장 성능이 좋은 `epoch_...ckpt`만 남기고 중간 단계는 삭제 후보입니다.

### [우선순위 3] 중복 및 백업 데이터
- **대상:** `git_recovery_backup` (**7.0G**)
  - 현재 레포지토리가 안정화되었다면 더 이상 필요하지 않을 수 있습니다.
- **대상:** `best_robovlms_mobile_model_epoch_1.pt` (**5.2G**)
  - 상위 디렉토리에 존재하는 단일 모델 파일입니다. 다른 곳에 백업이 있는지 확인 필요합니다.

### [우선순위 4] 데이터셋 관리
- **대상:** `ROS_action/` 하위 데이터셋
  - `basket_dataset_v2` (13G), `mobile_vla_dataset` (17G) 등.
  - 사용하지 않는 버전이나 중복된 데이터가 있는지 검토가 필요합니다.

## 📝 향후 조치 제안 (추후 요청 시)

1. **V3 정밀 정리:** V3 실험 폴더 내에서 `last.ckpt`를 제외한 모든 `epoch_*.ckpt` 삭제 시 약 **40~50GB** 확보 가능.
2. **V4 자동 관리:** Trainer 설정에서 `save_top_k=1` 등으로 변경하여 체크포인트 개수를 제한.
3. **아카이빙:** 정말 중요한 데이터는 외부 스토리지나 클라우드로 이동.
