## 4. Key Improvements Implemented
1.  **Redundant Normalization Fix**: Removed redundant `transforms.Normalize` in `inference_server.py`. The backbone already performs its own normalization. This restored visual sensitivity.
2.  **Action Smoothing (EMA)**: Added $0.6 \cdot current + 0.4 \cdot previous$ smoothing filter to prevent sudden action swaps (e.g., Forward to Backward).
3.  **Dynamic History Window**: Synchronized `head.history_len` to runtime `window_size` to prevent stale memory effects.
4.  **Forward-Ban Masking**: Implemented heavy logit penalty (-100.0) to force the model to explore secondary, trained diagonal strategies.

## 5. Conclusion & Recommendations
*   **Data Strategy**: The current 87% model is highly overfitted to straight trajectories. Future training must prioritize **Curve-Only filtering** or **Oversampling** of turn classes (L/R/FL/FR) to at least 50% of the total dataset.
*   **Heuristic Reliance**: In the current state, hard logit masking (F-Ban) is the most effective way to extract "intelligent" diagonal moves from the model.
*   **Next Steps**: Test "Oversampled" checkpoints if available in `runs/` and compare with the current F-Ban strategy.
