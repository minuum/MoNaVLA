## 2. 주요 기술적 문제 및 해결 방안
### [Issue 1] 학습 중단 (EinopsError)
- **원인:** `vla-driving` 브랜치에서 `third_party/RoboVLMs/main.py`의 로직이 수정되면서, `action_space`가 `discrete`인 경우에만 이산화 처리를 하도록 변경되어 V4 설정(`action_space: continuous`지만 `discrete_action: true`)과 충돌이 발생했습니다. 이로 인해 데이터셋은 연속형 액션(2차원)을 주었으나 모델은 이산형 레이블(1차원)을 기대하여 텐서 크기 불일치가 발생했습니다.
- **해결:** `main.py`의 `discrete_action` 판별 로직을 복구하여 설정 파일의 `discrete_action: true` 옵션이 정상 반영되도록 수정 완료했습니다.

### [Issue 2] 추론 시 모델 움직임 고정 (Action Sticking)
- **현상:** 실제 로봇 테스트 시 액션이 한 방향(주로 직진 또는 정지)으로 고정되는 현상이 보고되었습니다.
- **가설:** 
    1. **데이터셋 편향:** 새로 수집된 'Target-Only' 시나리오 데이터에 직진 비중이 너무 높아 모델이 직진만 학습했을 가능성.
    2. **LoRA 학습 부족:** 현재 Epoch 3까지 전이 학습된 상태로, 새로운 시나리오에 대한 특징 추출이 충분하지 않을 수 있음.
- **실행 계획:** 이산형 액션 레이블로 10 Epoch까지 학습을 완주한 후, 액션 출력의 분포를 다시 분석할 예정입니다.
