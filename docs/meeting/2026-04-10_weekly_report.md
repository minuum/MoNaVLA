## 3. Findings: 주요 기술적 발견

### 3.1 VLM 텍스트 이해 실패 확인
HF Cosmos 및 Google Robot 프리트레이닝 모델을 활용하여 텍스트-이미지 관계 이해 여부를 테스트한 결과, 모델이 **"go left", "go right" 등의 방향 명령어를 전혀 인식하지 못하고 의미 없는 값(링링링링)을 출력**하는 것을 확인했습니다.

- **원인 추정**: Google Robot 프리트레이닝 방식을 충분히 이해하지 않고 적용한 것이 원인일 가능성 있음
- **영향**: LoRA 파인튜닝 시도 이전에, VLM 백본 자체의 텍스트 이해 능력 부재가 근본 문제임을 확인

> [!IMPORTANT]
> 이번 발견으로 **"VLM이 텍스트 명령어를 이해하는가"** 를 최우선 검증 과제로 재설정하였습니다. 이 전제가 해결되지 않으면 Action Header 학습은 의미가 없습니다.

### 3.2 데이터셋 파이프라인 문제
바운딩 박스 오버레이 및 엔티티 분류(초록색: 바스켓/박스/컨테이너, 노란색: "gray" 포함)를 **50개 이미지**에 대해 완료하고 150개 데이터셋으로 학습을 진행하였으나, 구조적 문제가 발견되었습니다.

- **Train/Val 스플릿 오염**: 데이터셋이 에피소드 번호 순서대로 입력되어 validation과 training이 시간적으로 분리됨 → 스플릿 무효화
- **데이터셋 확장 필요**: 나머지 150개 이미지에 대한 바운딩 박스 작업 미완료
- **긍정적 진전**: H5 데이터셋에 XYZ 액션 값 추가 완료, Z축 데이터 신규 추가

### 3.3 LoRA 파인튜닝 한계
Open VLA에 LoRA를 적용하여 파인튜닝을 진행하였으나, 학습 결과가 기대에 미치지 못했습니다.

- **증상**: 텍스트 프롬프트 무시, 직진 방향으로만 예측 → 마지막 에폭에서 모든 포워드 액션이 좌측으로 잘못 수렴
- **구조적 결함 발견**: 기존 학습 코드에서 **CLIP 로스가 누락**되어 있었음 → 텍스트-비전 정렬이 전혀 이루어지지 않은 채 학습된 것

> [!NOTE]
> CLIP 로스 누락은 VLM이 텍스트를 학습 신호로 활용하지 못하게 만드는 치명적 결함입니다. 3.1의 문제와 함께 VLM 백본의 텍스트 이해 능력을 원점에서 재검증해야 합니다.
